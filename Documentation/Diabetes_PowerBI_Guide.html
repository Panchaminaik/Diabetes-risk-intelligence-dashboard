1
Load the CSV into Power BI
  1. Open Power BI Desktop → click Get Data → Text/CSV
  2. Browse to diabetes.csv → click Open
  3. In the preview dialog verify the first row is headers → click Transform Data (do NOT click Load yet)
  4. Power Query Editor will open — this is where we do all cleaning
💡 Always use Transform Data first. Never just "Load" — you need to add calculated columns.
2
Clean & Type-Cast Columns in Power Query
Set Correct Data Types

In Power Query, select each column header → right-click → Change Type:

ColumnSet Type To
id, age, glucose, blood_pressure, cholesterol, heart_rate, stress_level, steps_per_day, work_hours, insulinWhole Number
bmi, sleep_hours, water_intake_ltrDecimal Number
smoking, alcohol_intake, family_history, diabetesWhole Number (keep as 0/1)
gender, physical_activityText
Add a Simulated Date Column (for trend charts)

The dataset doesn't have dates but the dashboard needs monthly trends (Jul 2023 – May 2024). We'll assign a date row-by-row.
In Power Query → Add Column → Custom Column. Name it Date:

// Custom Column formula — assigns dates cycling across 11 months let months = { #date(2023,7,1), #date(2023,9,1), #date(2023,11,1), #date(2024,1,1), #date(2024,3,1), #date(2024,5,1) }, idx = Number.Mod([id] - 1, List.Count(months)) in months{idx}

Change the type of the new Date column to Date.

3
Add Calculated Columns in Power Query

Go to Add Column → Custom Column for each of the following:

A. Age Group Column
// Column Name: AgeGroup if [age] >= 18 and [age] <= 30 then "18-30" else if [age] >= 31 and [age] <= 45 then "31-45" else if [age] >= 46 and [age] <= 60 then "46-60" else "61+"
B. BMI Category Column
// Column Name: BMICategory if [bmi] < 18.5 then "Underweight" else if [bmi] < 25 then "Normal" else if [bmi] < 30 then "Overweight" else "Obese"
C. Risk Level Column (based on Glucose)
// Column Name: RiskLevel if [glucose] < 100 then "Low Risk" else if [glucose] < 140 then "Medium Risk" else if [glucose] < 180 then "High Risk" else "Critical Risk"
D. High Risk Flag Column
// Column Name: IsHighRisk if [glucose] > 180 and [bmi] > 30 then 1 else 0
E. Lifestyle Category (map physical_activity to 3 groups)
// Column Name: LifestyleGroup if [physical_activity] = "High" then "Active" else if [physical_activity] = "Medium" then "Moderate" else "Sedentary"
F. Residence Column (Urban/Rural — simulate from id)
// Column Name: Residence if Number.Mod([id], 10) < 6 then "Urban" else "Rural"
Why simulate? The dataset has no residence column. This 60/40 split matches the dashboard's 61.24%/38.76% distribution shown in the image.

Click Close & Apply when done. Your table now has 20 columns.

1
Create a Date Dimension Table

Go to Home → New Table in the DAX editor and paste:

// Table Name: DimDate DimDate = ADDCOLUMNS( CALENDAR(DATE(2023, 1, 1), DATE(2024, 12, 31)), "Year", YEAR([Date]), "Month", MONTH([Date]), "MonthName", FORMAT([Date], "MMM YYYY"), "Quarter", QUARTER([Date]), "MonthNum", MONTH([Date]) )
This gives you a proper calendar for time intelligence functions.

Create Dimension Tables (Optional but Recommended)
DimRiskLevel — for sorting Risk in the correct order

DimAgeGroup — for correct sort order
DimAgeGroup = DATATABLE( "AgeGroup", STRING, "AgeOrder", INTEGER, { { "18-30", 1 }, { "31-45", 2 }, { "46-60", 3 }, { "61+", 4 } } )
DimBMICategory — for heatmap sort order
DimBMICategory = DATATABLE( "BMICategory", STRING, "BMIOrder", INTEGER, { { "Underweight", 1 }, { "Normal", 2 }, { "Overweight", 3 }, { "Obese", 4 } } )

4
Create a Measures Table

Best practice is to keep all measures in one dedicated table:

  1. Go to Home → Enter Data
  2. Name it _Measures (the underscore keeps it at top of field list)
  3. Add one dummy column (e.g., "x" = 1) → click Load
  4. All your DAX measures will be created inside this table
★
All DAX Measures — Copy & Paste Each Into _Measures Table
To create a measure: Right-click _Measures table → New Measure → paste formula → Enter
A
Page 1 — Executive Overview KPIs
// ── KPI 1: Total Patients ────────────────────────────────────── Total Patients = COUNTROWS(diabetes) // ── KPI 2: Diabetes Patients ─────────────────────────────────── Diabetes Patients = CALCULATE(COUNTROWS(diabetes), diabetes[diabetes] = 1) // ── KPI 3: Diabetes Prevalence % ─────────────────────────────── Diabetes Prevalence % = DIVIDE([Diabetes Patients], [Total Patients], 0) * 100 // ── KPI 4: Average Glucose Level ─────────────────────────────── Avg Glucose = AVERAGE(diabetes[glucose]) // ── KPI 5: Average BMI ───────────────────────────────────────── Avg BMI = AVERAGE(diabetes[bmi]) // ── KPI 6: High-Risk Patients ────────────────────────────────── High Risk Patients = CALCULATE(COUNTROWS(diabetes), diabetes[IsHighRisk] = 1) High Risk % = DIVIDE([High Risk Patients], [Total Patients], 0) * 100
B
Page 1 — Charts & Gauge
// ── Gauge: Diabetes Risk Score (0-100 scale) ─────────────────── // Score based on avg glucose normalized to 0-100 Diabetes Risk Score = VAR AvgGlucose = AVERAGE(diabetes[glucose]) VAR MinG = 50 VAR MaxG = 300 RETURN ROUND((DIVIDE(AvgGlucose - MinG, MaxG - MinG, 0)) * 100, 0) // ── Trend Chart: Patients by Age Group ───────────────────────── Patients by Age = COUNTROWS(diabetes) Diabetes Prevalence by Age = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[diabetes] = 1), COUNTROWS(diabetes), 0 ) * 100
C
Page 2 — Risk Analysis KPIs
// ── KPI: High Glucose Rate (Glucose > 180) ───────────────────── High Glucose Rate % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[glucose] > 180), [Total Patients], 0 ) * 100 // ── KPI: Obesity Rate (BMI > 30) ─────────────────────────────── Obesity Rate % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[bmi] > 30), [Total Patients], 0 ) * 100 // ── KPI: Family History Risk % ──────────────────────────────── Family History Risk % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[family_history] = 1), [Total Patients], 0 ) * 100 // ── KPI: Smoking Impact Score ────────────────────────────────── // Measures how much more diabetes prevalence smokers have vs non-smokers Smoking Impact Score = VAR SmokerDiabetes = CALCULATE( DIVIDE(COUNTROWS(FILTER(diabetes, diabetes[diabetes]=1 && diabetes[smoking]=1)), COUNTROWS(FILTER(diabetes, diabetes[smoking]=1)), 0) ) VAR NonSmokerDiabetes = CALCULATE( DIVIDE(COUNTROWS(FILTER(diabetes, diabetes[diabetes]=1 && diabetes[smoking]=0)), COUNTROWS(FILTER(diabetes, diabetes[smoking]=0)), 0) ) RETURN ROUND((SmokerDiabetes - NonSmokerDiabetes) * 100, 2)
D
Page 2 — Risk Segmentation & Waterfall
// ── Risk Segments (for Funnel / Bar chart) ───────────────────── Low Risk Count = CALCULATE(COUNTROWS(diabetes), diabetes[RiskLevel] = "Low Risk") Medium Risk Count = CALCULATE(COUNTROWS(diabetes), diabetes[RiskLevel] = "Medium Risk") High Risk Count = CALCULATE(COUNTROWS(diabetes), diabetes[RiskLevel] = "High Risk") Critical Risk Count = CALCULATE(COUNTROWS(diabetes), diabetes[RiskLevel] = "Critical Risk") // ── Waterfall: Risk Factor Impact ────────────────────────────── // Base diabetes rate with NO risk factors Base Diabetes Rate = CALCULATE( DIVIDE(COUNTROWS(FILTER(diabetes, diabetes[diabetes]=1)), COUNTROWS(diabetes), 0), diabetes[glucose] <= 140, diabetes[bmi] <= 30, diabetes[family_history] = 0, diabetes[smoking] = 0 ) * 100 // Impact of High Glucose alone High Glucose Impact = CALCULATE( DIVIDE(COUNTROWS(FILTER(diabetes, diabetes[diabetes]=1 && diabetes[glucose] > 140)), COUNTROWS(FILTER(diabetes, diabetes[glucose] > 140)), 0) ) * 100 - [Base Diabetes Rate] // ── High-Risk Patients Over Time (line chart) ────────────────── High Risk Over Time = CALCULATE([High Risk Patients])
E
Page 3 — Demographics KPIs
// ── KPI: Active Lifestyle % ──────────────────────────────────── Active Lifestyle % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[LifestyleGroup] = "Active"), [Total Patients], 0 ) * 100 // ── KPI: Sedentary Lifestyle % ──────────────────────────────── Sedentary Lifestyle % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[LifestyleGroup] = "Sedentary"), [Total Patients], 0 ) * 100 // ── KPI: Male Diabetes % ────────────────────────────────────── Male Diabetes % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[gender] = "Male", diabetes[diabetes] = 1), CALCULATE(COUNTROWS(diabetes), diabetes[gender] = "Male"), 0 ) * 100 // ── KPI: Female Diabetes % ──────────────────────────────────── Female Diabetes % = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[gender] = "Female", diabetes[diabetes] = 1), CALCULATE(COUNTROWS(diabetes), diabetes[gender] = "Female"), 0 ) * 100 // ── Physical Activity Combo Chart ───────────────────────────── Patients by Lifestyle = COUNTROWS(diabetes) Diabetes Rate by Lifestyle = DIVIDE( CALCULATE(COUNTROWS(diabetes), diabetes[diabetes] = 1), COUNTROWS(diabetes), 0 ) * 100 // ── Gender counts (for donut chart) ─────────────────────────── Male Patients = CALCULATE(COUNTROWS(diabetes), diabetes[gender] = "Male") Female Patients = CALCULATE(COUNTROWS(diabetes), diabetes[gender] = "Female") // ── Family History (for donut chart) ────────────────────────── With Family History = CALCULATE(COUNTROWS(diabetes), diabetes[family_history] = 1) No Family History = CALCULATE(COUNTROWS(diabetes), diabetes[family_history] = 0) // ── Residence (for donut chart) ─────────────────────────────── Urban Patients = CALCULATE(COUNTROWS(diabetes), diabetes[Residence] = "Urban") Rural Patients = CALCULATE(COUNTROWS(diabetes), diabetes[Residence] = "Rural")

Funnel/Bar Chart — Risk Segmentation

Power BI's built-in Funnel chart works well here. Alternatively use a stacked bar.

  1. Insert Funnel Chart visual
  2. Category: DimRiskLevel[RiskLevel] (sorted by RiskOrder descending for funnel shape)
  3. Values: [Total Patients] (it will filter per context)
  4. Actually use a calculated table approach: Create a new table:
    RiskSummary = SUMMARIZE( diabetes, diabetes[RiskLevel], "PatientCount", COUNTROWS(diabetes), "Prevalence", DIVIDE(COUNTROWS(FILTER(diabetes, diabetes[diabetes]=1)), COUNTROWS(diabetes), 0) * 100 )
  5. Funnel: Category = RiskSummary[RiskLevel], Values = RiskSummary[PatientCount]
  6. Colors: Critical=Red, High=Orange, Medium=Yellow, Low=Green
  7. Add data labels showing count + % label on each bar

Heatmap — Age Group vs BMI Category
⚠️ Power BI has no native heatmap. Use one of these approaches:
Option A: Matrix Visual (Recommended)
  1. Insert Matrix visual
  2. Rows: DimAgeGroup[AgeGroup]
  3. Columns: DimBMICategory[BMICategory]
  4. Values: [Total Patients]
  5. Format → Cell elements → Background color → Conditional formatting → Color scale
  6. Set Min color = white, Max color = #EF4444 (red)
  7. This creates a heatmap effect with color intensity = patient count
Option B: Download "Heatmap" from AppSource
  1. Visualizations pane → Get more visuals → Search "Heatmap"
  2. Install "Heatmap by MAQ Software" (free)
  3. Set X=BMICategory, Y=AgeGroup, Values=patient count
